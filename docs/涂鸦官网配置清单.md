# 涂鸦官网配置清单

本清单对应当前工程的 TuyaOpen 设备端、微信云函数和小程序代码。请先完成产品 DP 配置，再启动新编译的设备端程序。

## 1. 修改现有功能点

在涂鸦开发者平台进入当前产品的“功能定义”。保留现有 DP ID 和标识符，只修改展示名称或单位。

| DP ID | 标识符 | 修改内容 |
|---|---|---|
| 105 | `air_quality_ppm` | 名称改为“空气质量相对值”，单位改为 `%`；数值范围 `0~100`、步长 `1`、倍数 `0` |
| 106 | `env_status` | 名称改为“传感器状态”；枚举保持 `normal/partial_error/error/unknown` |
| 125 | `last_update` | 名称改为“品质更新时间”；字符串、只上报 |
| 148 | `device_status` | 确认枚举为 `idle/running/error/offline/unknown`，只上报 |

不要删除并重建这些功能点，否则 DP ID 变化后设备端数字 ID 会失配。

## 2. 新增检测执行状态

新增三个自定义功能点，全部设置为“只上报”。标识符、数据类型和枚举值必须完全一致。

| 期望 DP ID | 标识符 | 类型 | 配置 |
|---|---|---|---|
| 150 | `detect_request_id` | 字符串 | 最大长度至少 `64` |
| 151 | `detect_status` | 枚举 | `idle,starting,running,stopping,snapshotting,snapshot_done,failed` |
| 152 | `detect_result` | 字符串 | 最大长度至少 `255` |

当前 C 设备端使用 150、151、152。如果平台自动分配的实际 ID 不同，应修改：

`iot/TuyaOpen/apps/tuya_cloud/fruit_quality_cloud/src/fruit_quality_bridge.c`

中的 `DP_DETECT_REQUEST_ID`、`DP_DETECT_STATUS`、`DP_DETECT_RESULT`，然后重新执行 `tos.py build`。Python 代理和微信云函数使用标识符，不依赖数字 ID。

已有 `detect_cmd` 保持 DP 146、可下发可上报，枚举保持 `idle/start/stop/snapshot`。

## 3. 面板配置

在产品面板中：

- 显示 `detect_status`，名称为“检测状态”。
- 显示 `detect_result`，名称为“执行结果”。
- `detect_request_id` 用于程序确认命令，面板中可以隐藏。
- `detect_cmd` 继续作为开始、停止、抓拍控制项。
- `device_status` 作为整机状态展示，不再固定显示运行中。

保存功能定义并重新发布产品配置。若平台提示已发布产品不能直接修改单位，可只修改面板显示单位，不要删除原 DP。

## 4. 云端与设备更新顺序

1. 在涂鸦产品中完成上述 DP 配置并发布。
2. 确认云项目仍已关联该产品和设备，并具备设备状态、设备影子和命令下发权限。
3. 重新部署微信云函数 `wechat_miniapp/cloudfunctions/tuyaDevice`。
4. 上传或预览更新后的微信小程序。
5. 使用新构建的 `fruit_quality_cloud_0.1.0.elf` 重启设备端 IoT 进程。

新设备端产物路径：

`iot/TuyaOpen/apps/tuya_cloud/fruit_quality_cloud/dist/fruit_quality_cloud_0.1.0/fruit_quality_cloud_0.1.0.elf`

## 5. 联调检查

按以下顺序检查设备影子属性变化：

1. 下发 `detect_cmd=start`。
2. `detect_request_id` 应生成新值。
3. `detect_status` 应从 `starting` 变为 `running`。
4. 下发 `detect_cmd=snapshot`，成功时状态应变为 `snapshot_done`，`detect_result` 返回本地文件名；无画面时应变为 `failed`。
5. 下发 `detect_cmd=stop`，最终状态应为 `idle`。
6. 在 Qt 本地切换传送带档位或 LED 亮度，设备影子中的对应 DP 应同步变化。
7. 环境上报后，`last_update` 不应再变化；它只由品质结果更新。

小程序只有收到最终 `detect_status` 后才显示命令成功，不再把云端接收命令误判为设备执行成功。
