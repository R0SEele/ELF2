# 微信小程序环境历史配置

环境历史由微信云函数 `tuyaDevice` 每 5 分钟读取一次涂鸦设备影子，并写入微信云数据库。小程序查询最近 7 天的云端记录，与当前手机上的实时采样合并后绘制趋势。

## 1. 创建数据库集合

在微信开发者工具进入当前云环境 `cloud1-d3gawfd5o88014f88`：

1. 打开“云开发” -> “数据库”。
2. 新建集合，名称必须为 `environment_history`。
3. 集合权限选择“仅管理员可读写”或等价的禁止小程序端直接读写权限。云函数仍可正常访问。
4. 新建组合索引：`device_id` 升序、`timestamp` 升序。

集合不需要手工添加字段，第一次采集成功后会自动生成记录。

## 2. 重新部署云函数

右键本地 `cloudfunctions/tuyaDevice`，选择：

`上传并部署：云端安装依赖（不上传 node_modules）`

本次新增 `wx-server-sdk` 依赖，不能只上传单个 `index.js`。部署后确认原有环境变量仍然存在：

- `TUYA_ENDPOINT`
- `TUYA_CLIENT_ID`
- `TUYA_CLIENT_SECRET`
- `TUYA_DEVICE_ID`
- `APP_ACCESS_TOKEN`（使用访问口令时）

函数超时时间建议设置为至少 20 秒。

## 3. 检查定时触发器

部署目录中的 `config.json` 声明了定时触发器：

- 名称：`environmentHistoryCollector`
- 类型：定时触发器
- Cron：`0 */5 * * * * *`
- 周期：每 5 分钟一次

如果部署后云函数的“触发管理”中没有该触发器，按上述参数手工创建。

## 4. 手工验证

在云函数测试控制台先执行一次采集：

```json
{
  "action": "collect_environment"
}
```

如果配置了 `APP_ACCESS_TOKEN`，测试事件还要加入同值的 `access_token`。

成功结果应包含：

```json
{
  "ok": true,
  "source": "cloud_history",
  "recorded": true
}
```

随后检查 `environment_history` 集合，应出现包含 `device_id`、`timestamp`、`values` 和 `env_status` 的记录。

再执行历史查询：

```json
{
  "action": "environment_history",
  "range_minutes": 60
}
```

成功结果中的 `points` 应至少有一条数据。

## 5. 小程序验收

重新编译小程序并进入“环境趋势分析”页。曲线下方的数据来源应显示“云端历史”和实际数据起止时间。如果显示“本机缓存”，检查云函数日志中是否存在集合不存在、索引缺失、环境变量缺失或涂鸦接口请求失败。

云端记录保留 7 天，当前小程序页面可切换 10 分钟、1 小时、24 小时和 7 天。设备没有上报新环境时间戳时不会重复写入旧记录。

## 6. 导入 RK3588 现有历史

工程提供了转换脚本，将 `sensor_realtime.csv` 最近 7 天数据转换成微信云数据库 JSON Lines：

```bash
python3 /home/elf/projects/cloud/tuya_proxy/export_environment_history.py
```

输出文件：

`/home/elf/projects/datas/import/environment_history_7d.json`

在 `environment_history` 集合中选择“导入”，文件类型选择 JSON，上传该文件。每条记录使用确定性 `_id`，同一文件不要重复导入。导入完成后重新进入环境趋势页即可看到历史数据。
