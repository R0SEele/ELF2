## 微信小程序

小程序名 **芒果质检助手 / 芒果智检分拣终端**（`projectname: mango-quality-tuya`，`appid: wx12bf3c4c73e5b3f2`），通过微信云开发云函数 `tuyaDevice` 调用涂鸦云 API，读取设备影子属性并下发控制命令，实现远程监控与操控。

### 3. 全局逻辑（`app.js`）

`DEFAULT_CLOUD_ENV = "cloud1-d3gawfd5o88014f88"`。

`globalData`：`apiBase`、`cloudEnv`、`useCloud`（默认 false，onLaunch 启用云）、`accessToken`。

`onLaunch()`：读本地 `accessToken`；有 `apiBase` 存储则 `configureEndpoint(stored,false)`，否则 `enableCloud(DEFAULT_CLOUD_ENV,false)`。

**核心方法**：
- `normalizeCloudEnv`：剥 `cloud://`/`env:` 前缀。
- `configureEndpoint(value,persist)`：判断云环境还是本地地址，分别 `enableCloud`/`disableCloud`。
- `enableCloud(envId,persist)`：`wx.cloud.init({env,traceUser:true})`，`useCloud=true`，`apiBase=cloud://<env>`。
- `disableCloud(apiBase,persist)`：本地模式，`apiBase` 默认回退 `http://127.0.0.1:8765`。
- `setAccessToken(token)`：设置并持久化。
- **`request(path,options)`**：
  - **云模式**：`path` 含 `/api/device/status` → action `status`，否则 `command`。`wx.cloud.callFunction({name:"tuyaDevice",data})`；status 时 `data={action,access_token}`，command 时 `data={action,command:options.data,access_token}`。判 `res.result.ok !== false`。
  - **本地模式**：`wx.request`，`url=apiBase+path`，超时 8000ms，有令牌带头 `X-Access-Token`。判 HTTP 2xx 且 `res.data.ok !== false`。
- `requestDeviceStatus()` → `request("/api/device/status")`。
- `sendDeviceCommand(code,value)` → `request("/api/device/command",{method:"POST",data:{code,value}})`。

5. **批次统计卡**：总数 + 堆叠进度条（A/B/剔除 `aPct/bPct/rejectPct`）+ 图例。
6. **设备控制卡**：传送带/传送速度/分拣位置三个分段选择器；💡补光开关（选中 `#2fb37a`）；亮度 slider(0–100 步1)；🤖自动分拣开关；检测动作按钮组。
7. **页脚**：`RK3588 · 端侧 AI 视觉质检系统`。

**控制选项（`optionSets`）**：`conveyor_cmd`(stop/forward/reverse)、`conveyor_speed`(slow/medium/fast)、`sorter_position`(left/center/right)、`detect_cmd`(start/stop/snapshot)。

**语义配色映射**：
- `GRADE_CLASS`：a→ok, b→warn, c→info, reject→danger, unknown→muted。
- `CHANNEL_CLASS`：sales→ok, ripen→warn, process/recheck→info, reject→danger, unknown→muted。
- `ENV_CLASS`：normal→ok, partial_error→warn, error→danger, unknown→muted。
- `ENV_ICON`：temperature_c 🌡️、humidity_rh 💧、co2_ppm 🫧、light_lux ☀️、air_quality_ppm 🍃。

**数据获取与轮询**：
- `onShow` 启动、`onHide`/`onUnload` 停止。`startPolling()` 先 `refreshStatus()` 再 `setInterval(3000)`（每 3s）。
- **并发锁**：`_inflight`+`_inflightAt` 时间戳；上一请求 10s 未返回强制放行。
- **数据装配**：`app.requestDeviceStatus()` → `itemMap(data.groups.*)` → `overview`（quality 组）、`environment`（过滤 env_status 加 icon）、`deviceStatus`（`deviceClass`：running→ok/error→danger/offline→muted/idle→info）、`cloudMode=data.source==="tuya"`、`lastUpdated=formatTime(updated_at)`、`batch`（`buildBatch` 算占比）。
- **乐观锁**：下发命令后 `controlLockUntil=Date.now()+4000`，4s 窗口内保留本地控件状态防回弹。
- **失败**：catch 置 `connected:false`、`deviceStatus:"未连接"`，有 `err.error` 则 toast。

**命令下发** `sendCommand(code,value)`：设乐观锁 → `app.sendDeviceCommand` → 成功 `refreshStatus()`。事件：`onLedSwitch/onBrightnessChange/onAutoSortChange/onConveyorTap/onSpeedTap/onSorterTap/onDetectTap`。

**保存连接** `saveConnection`：校验 `app.configureEndpoint`，保存令牌，放行旧请求后刷新。

### 5. 质检详情页 `pages/quality/quality`

展示当前单果完整质检明细。UI：topbar（标题 + `云端同步/未连接·lastUpdated` + 刷新）、headline（编号/等级/通道/结论）、成熟度 section（metric-grid）、糖度与腐烂 section、视觉识别 section、数据状态 section。

JS：同首页 3s 轮询+并发锁（onShow/onHide）。`refreshStatus()` 取 `payload.groups.quality`，`findItems(quality,[codes])` 分组：`headline`(mango_id/quality_grade/suggested_channel/final_status)、`maturity`(maturity_label/score/confidence)、`sugar`(sugar_label/score)、`rot`(rot_status/score)、`vision`(yolo_label/confidence)、`data`(data_status/last_update)。`onRefreshTap` 手动刷新；`formatTime` HH:MM:SS。

### 6. 云函数 `tuyaDevice`

**依赖**：`name:tuyaDevice, version:1.0.0`，**无第三方依赖**，仅 Node 内置 `crypto`（签名）+ `https`。

**入口 `exports.main(event)`**：① 鉴权（设 `APP_ACCESS_TOKEN` 且 `event.access_token` 不匹配 → `{ok:false,error:"访问口令错误"}`）；② 路由（`status`→`getStatus()`，`command`→`sendCommand(event.command||event)`，否则 `Unknown action`）。异常统一 catch。

**配置 `config()`**：环境变量 `TUYA_ENDPOINT`(默认 openapi.tuyacn.com)、`TUYA_CLIENT_ID`、`TUYA_CLIENT_SECRET`、`TUYA_DEVICE_ID`；缺必需项抛错。

**签名 `signHeaders`（HMAC-SHA256）**：`bodyHash=sha256(body)`；`stringToSign=METHOD\n bodyHash\n\n pathWithQuery`；`signSource=clientId+token+timestamp+nonce+stringToSign`；`sign=HMAC-SHA256(signSource,clientSecret).toUpperCase()`；头 `client_id,sign,t,nonce,sign_method:"HMAC-SHA256"`（有 token 加 access_token）。`urlPath()` query 键排序+URL 编码。

**令牌 `token()`**：内存缓存 `cachedToken`+`cachedTokenExpireAt`（提前 60s）；`GET /v1.0/token?grant_type=1`；`tuyaAuthedRequest()` 遇 1010/1011/1012/1013 清缓存重试一次。

**API 调用**：
- 读状态 `getStatus()` → `GET /v2.0/cloud/thing/{deviceId}/shadow/properties`，取 `result.properties`。
- 下发 `sendCommand()` → `POST /v2.0/cloud/thing/{deviceId}/shadow/properties/issue`，body `{properties:{[code]:value}}`。

**DP_META 与分组**：与 tuya_proxy 一致（见 [05-Qt上位机与云端涂鸦IoT.md](05-Qt上位机与云端涂鸦IoT.md)）。`normalizeValue`：bool→开启/关闭；enum→中文 display；value→除 `10^scale` 拼单位；string→文本（空 `--`）。`buildStatusPayload` 产出 `{ok,source:"tuya",device_id,updated_at,status,items[],groups{},group_labels}`。`validateCommand`：仅 writable；类型校验；`led_brightness` clamp 0–100。

### 附：数据流与契约关系小结

- **视觉链路**：YOLO11(`yolo11.yaml`) → `mango_object_realtime.*`、`vision_color_realtime.csv` → 融合 → `mango_quality_realtime.*`、`mango_quality_history.csv` → 批次 `mango_batch_summary.*`。
- **环境链路**：传感器(`sensors.yaml`) → `sensor_realtime.csv`；补光(`led.yaml`) → `led_state.csv`。
- **执行链路**：融合决策 → 舵机(`servo.yaml`) → `mango_sorter_events.csv`；传送带(`motor.yaml`)。
- **云/远程链路**：设备影子(涂鸦, `tuya_cloud.json`) ↔ 云函数 `tuyaDevice`/`tuya_proxy` ↔ 微信小程序；本地代理镜像 `tuya_proxy_control_state.json`。
- 小程序 `DP_META` 分组（quality/environment/batch/control/device）与上述 CSV/JSON 字段、`servo/motor/led.yaml` 控制项构成端到端一致的数据契约。
