# 01 · 系统架构总览

本篇说明整机的分层架构、进程编排、数据总线契约与跨模块协作关系。各模块的实现细节见对应分册（02~06）。

---

## 1. 架构原则

整个系统遵循两条核心设计原则：

1. **以文件为数据总线（File as message bus）**
   每个采集/推理程序把结果**原子写入**（先写 `*.tmp` 再 `os.replace`）`datas/csv/` 下的 CSV/JSON。消费方（Qt、小程序、云固件、语音助手）只读文件、不直接调用彼此。好处：模块解耦、可独立启停、可离线回放、天然容错。

2. **以子进程为控制通道（Subprocess as control channel）**
   上层（Qt、tuya_proxy、TuyaOpen 固件）通过 `subprocess`/`QProcess`/`system()` 拉起 Python 硬件脚本（`conveyor_cli.py` / `sorter.py` / `ws2812b.py`）下发控制命令，并以命令行参数传参。三条上层链路调用的是**同一批脚本、同一套速度档（0.10/0.13/0.16 m/s）**，保证云端与本地行为一致。

---

## 2. 分层职责

| 层 | 组成 | 输入 | 输出 |
|----|------|------|------|
| 硬件驱动层 | camera/spectrum/sensors/led/motor/servo | 物理外设（I2C/SPI/UART/PWM/ADC） | 传感读数、执行动作 |
| 数据契约层 | `datas/csv/*`、`config/*` | 驱动/算法写入 | 全系统共享状态 |
| 软件算法层 | YOLO11 推理、fusion 融合、auto_sorter | 摄像头帧、光谱、传感 CSV | 质检结论、分拣决策 |
| 云端接入层 | tuya_proxy、TuyaOpen 固件 | 本地 CSV/JSON、云端 DP | 上云上报、下发控制 |
| 交互展示层 | Qt HMI、语音助手、微信小程序 | 质检/环境/批次 CSV、设备状态 | 图表、语音、远程操控 |

## 3. 进程编排（Qt 作为总控）

Qt 上位机 `MainWindow` 在进入工作页时统筹拉起后端进程，退出时复位硬件（`shutdownHardware()` 幂等）：

```
MainWindow 构造
 ├─ startTuyaIotProcess()            拉起涂鸦固件 ELF（若未运行）
 └─ m_iotStatusTimer (5s)            周期探测 openapi.tuyacn.com 网络

showWorkPage()（进入工作页）
 ├─ startSensorProcess()             python3 csv_logger.py --interval 5
 │   └─ m_sensorTimer (5s)           → refreshSensorData() 刷新 6 张环境卡
 ├─ startCameraProcess()             python3 camera_detect.py --qt-stream --parent-pid
 │   └─ readCameraFrames()           读 4字节长度头+JPEG 帧流显示视频
 └─ startMangoQualityProcess()       python3 mango_quality_cli.py
     └─ m_mangoQualityTimer (1s)     → refreshMangoQualityData() + refreshBatchStatsData()
```

- **单线程 GUI 模型**：无 QThread，全部并发靠 `QProcess`（子进程）+ `QTimer`（事件循环）+ `QSocketNotifier`/`QNetworkReply`（异步 IO）。
- **父进程守护**：`camera_detect.py`、`csv_logger.py`、`mango_quality_cli.py` 均支持 `--parent-pid`，父进程消失即自杀，避免 Qt 崩溃后残留僵尸进程。
- **信号安全退出**：`main.cpp` 用 self-pipe trick 捕获 SIGINT/SIGTERM/SIGHUP，退出前统一复位硬件。

## 5. 配置体系（`config/`）

集中式 YAML/JSON 配置，各驱动实现 **命令行参数 > YAML 配置 > 代码默认值** 三级回退（通用函数 `_cfg_or_default`）。

| 文件 | 覆盖模块 |
|------|----------|
| `yolo11.yaml` | YOLO11 RKNN 推理参数 |
| `sensors.yaml` | SHT30/SCD40/GY-302/MQ-135/AS7341 |
| `led.yaml` | WS2812B 补光灯 |
| `motor.yaml` | 步进电机/传送带 |
| `servo.yaml` | 舵机/自动分拣通道映射 |
| `tuya_cloud.json` | 涂鸦云 endpoint/client_id/device_id（非机密） |
| `tuya_cloud_secrets.json` | client_secret（git 忽略） |

## 6. DP（数据点）语义统一

涂鸦云的 DP 定义在三处保持一致，是端到端数据契约的枢纽：

- `cloud/tuya_proxy/tuya_proxy.py` 的 `DP_META`（字符串 code，35 个 DP）
- `wechat_miniapp/cloudfunctions/tuyaDevice/index.js` 的 `DP_META`（同上）
- `iot/TuyaOpen/apps/.../fruit_quality_bridge.c`（数字 DP ID 101~149，与字符串 code 一一对应）

分组：`device 设备` / `quality 当前芒果` / `environment 环境` / `batch 批次` / `control 控制`。控制组（可写）：`conveyor_cmd/conveyor_speed/sorter_position/led_switch/led_brightness/detect_cmd/auto_sort_enable`。

## 7. 跨模块协作时序（一次完整质检）

```
1. 芒果随传送带进入相机视野
2. camera_detect.py: YOLO11 逐帧检测 → MangoTracker 多帧关联跟踪
3. 芒果越过计数线 → 多帧投票得稳定标签+颜色特征 → 写 mango_object_realtime.*
4. collect.py: AS7341 采集该芒果光谱 → 写 spectrum_quality_samples.csv
5. mango_quality_cli.py 轮询(0.5s): read_latest_object + read_latest_spectrum
6. assess_mango_quality: 三路成熟度评分加权 + 腐烂/糖度评分 → 分级 + 建议通道
7. 写 mango_quality_realtime.* / history / batch_summary
8. auto_sorter.sort_once: 通道→舵机档位 → 延时 delay_s → sorter.py <pos> → 舵机PWM
9. 写 mango_sorter_events.csv；舵机复位到中心
10. voice_assistant 检测到新结果 → DeepSeek 生成播报 → TTS 发声
11. Qt 刷新图表；固件/proxy 按 DP 上报涂鸦云 → 小程序显示
```
