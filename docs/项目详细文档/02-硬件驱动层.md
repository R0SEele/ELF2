# 02 · 硬件驱动层技术文档

## 概述

本篇覆盖 `/home/elf/projects/src/hardware/` 下的全部硬件驱动子系统，运行平台为 RK3588 边缘计算板（工程内标识为 "ELF RK3588"），操作系统为 Debian/Ubuntu 系（`__pycache__` 显示 CPython 3.10）。硬件层分为 6 个模块：

| 模块 | 目录 | 硬件对象 | 主要接口 |

## led 模块（WS2812B 白光照明）

### 文件：`led/ws2812b.py`

**作用**：把 WS2812B 灯环当作纯白光补光灯控制，通过 **SPI 位模拟（bit-banging over SPI MOSI）** 生成 WS2812B 时序。命令行接口 `set`/`off`。

**依赖库**：`argparse`、`csv`、`fcntl`、`os`、`pyyaml`。无第三方 LED 库，纯 SPI ioctl 实现。

#### 硬件接口/协议（精确值）

- SPI 设备：`/dev/spidev4.0`（`spi_bus=4`, `spi_device=0`）——WS2812B 的 DIN 接 **SPI4 MOSI**。
- SPI 速率：`spi_speed_hz = 2_400_000`（2.4 MHz）。此速率下 1 个 SPI 位 ≈ 0.4167 µs，用 3 个 SPI 位编码 1 个 WS2812B 位。
- SPI ioctl 常量：
  - `SPI_IOC_WR_MODE = 0x40016B01` → 设置 SPI 模式 0（写入 `bytes([0])`）
  - `SPI_IOC_WR_BITS_PER_WORD = 0x40016B03` → 8 bit/word
  - `SPI_IOC_WR_MAX_SPEED_HZ = 0x40046B04` → 写入 `spi_speed_hz.to_bytes(4,"little")`
- SPI 写入用 `os.open(device, O_WRONLY)` + 循环 `os.write` 直至写完整帧。

#### WS2812B 编码算法

- 位模式（配置 `bit0_pattern="100"`, `bit1_pattern="110"`）：`_expand_byte()` 从 MSB(bit7) 到 LSB(bit0) 遍历，每位展开为 3 个 SPI 位（0→`100`，1→`110`），符合 WS2812B 归零码时序。
- 颜色顺序 `color_order="GRB"`：`_white_frame()` 对每颗灯按 G、R、B 顺序发送同一 `value`（白光，R=G=B）。校验 `sorted(color_order)==["B","G","R"]`。
- 帧尾复位：`bits.extend([0]*240)`（240 个 0 位 = 80 字节，在 MOSI 上产生 >50 µs 低电平复位）。
- `_pack_bits()`：把位流每 8 位打包成 1 字节，末尾不足则左移补齐（`current << (8-used)`）。

#### 亮度计算公式

```
brightness_pct = clamp(brightness_pct, 0..100)
brightness_pct = min(brightness_pct, max_brightness_pct)   # 上限保护
value = round(255.0 * brightness_pct / 100.0)  if enabled else 0
value = clamp(value, 0..255)
```

#### 关键函数

| 函数 | 参数 | 返回 |
|------|------|------|
| `_expand_byte(value, bit0_pattern, bit1_pattern)` | 字节值+两种位模式 | SPI 位列表 |
| `_pack_bits(bits)` | 位列表 | `bytearray` |
| `_white_frame(count, value, color_order, bit0, bit1)` | 灯数、白光值、顺序、位模式 | 完整帧字节 |
| `_apply_white(cfg, brightness_pct, enabled, count_override)` | 配置+亮度+使能 | 无（写 SPI） |
| `_write_state(state_file, brightness_pct, enabled)` | 状态文件路径 | 写 CSV |

**异常**：`WS2812BError`（模式非二进制、count≤0、颜色顺序错、设备不存在、无权限、SPI 写失败）。

#### 数据输出（CSV）

`_write_state()` 写 `state_file`（配置 `datas/csv/led_state.csv`）：列 `enabled`（0/1）、`brightness_pct`（1 位小数）；每次覆盖写（模式 `"w"`），只保存最新一行状态。

### 文件：`led/setup_led_permissions.sh`

LED 权限脚本，实为**转发器**——`exec` 调用 `sensors/setup_sensor_permissions.sh "$@"`（WS2812B 用 SPI，spi 组权限由 sensors 脚本统一配置）。

### 配置项 `config/led.yaml`

`led.ws2812b`：`driver: spi`、`device: /dev/spidev4.0`、`spi_bus:4`、`spi_device:0`、`spi_speed_hz:2400000`、`bit0_pattern:"100"`、`bit1_pattern:"110"`、`color_order:GRB`、`count:90`（灯环 90 颗）、`default_brightness_pct:40`、`max_brightness_pct:100`、`state_file:.../led_state.csv`、`sensor_csv:.../sensor_realtime.csv`（后者供其他消费方保留，ws2812b.py 未读）。

## motor 模块（步进电机 / 传送带控制）

包结构：`__init__.py` 导出 `MotorControllerError, MotorLimits, ZDTMotorController, ProtocolError, ZDTV2Protocol, SerialBusError, SerialConfig, UARTBus`。

### 文件：`motor/serial_bus.py`（UART 底层总线）

**作用**：线程安全的串口读写封装。**依赖库**：`pyserial`、`threading`、`time`。

#### `SerialConfig`（dataclass，默认值即硬件接口）

`port="/dev/ttyS9"`, `baudrate=115200`, `bytesize=EIGHTBITS`, `parity=PARITY_NONE`, `stopbits=STOPBITS_ONE`（即 **8N1**）, `timeout_s=0.2`, `write_timeout_s=0.2`。

#### `UARTBus` 方法

| 方法 | 说明 |
|------|------|
| `exchange(req, read_size, read_timeout_s)` | 清输入缓冲→写→flush→循环读满 `read_size` 字节，超时抛 `SerialBusError` |
| `exchange_min(req, min_size, read_timeout_s, strip_echo=True)` | 读“至少 `min_size`”字节；用 20 ms 空闲窗口尽量收全一帧；回显以 `req` 开头则剥除（half-duplex） |
| `write_only(req)` | 只写不读（`wait_ack=False` 时用） |
| `probe(req, read_timeout_s)` | 发送后在超时窗内收集所有字节（诊断用） |
| `close()` | 关闭串口 |

所有 I/O 用 `threading.Lock` 串行化。

### 文件：`motor/protocol.py`（ZDT_X/Emm 串口协议）

**作用**：实现 ZDT_X V2 与 Emm_V5 两种步进驱动器协议的帧封装与解析。**异常**：`ProtocolError`。

#### 帧格式（`Frame` dataclass）

```
帧 = [address(1B)] + [command(1B)] + payload + [tail]
tail 固定 = 0x6B     （单字节地址 + 固定尾字节，无 CRC）
```

`to_bytes()`：`bytes([addr&0xFF, cmd&0xFF]) + payload + bytes([tail&0xFF])`。

#### 命令码常量（`ZDTV2Protocol`）

读类：`CMD_READ_VERSION=0x1F`、`CMD_READ_ENCODER_RAW=0x30`、`CMD_READ_SPEED=0x35`、`CMD_READ_POSITION=0x36`、`CMD_READ_STATUS=0x3A`。
控制类：`CMD_ENABLE=0xF3`、`CMD_SPEED_MODE=0xF6`、`CMD_POS_MODE=0xFD`、`CMD_STOP=0xFE`。

#### 命令封装细节（大端序 big-endian）

- **使能** `cmd_enable`：`addr + F3 + AB + 使能状态(0x01/0x00) + 同步标志 + 6B`（0xAB 固定功能字节）。
- **停止** `cmd_stop`：`addr + FE + 98 + 0x00 + 6B`。
- **速度模式** `cmd_speed_mode`（按 `variant` 分支）：
  - `zdt_v2`：`addr + F6 + 符号(dir&0x01) + 速度斜率u16(RPM/s) + 速度u16(0.1RPM,即speed×10) + 同步 + 6B`。`speed_x10=round(abs(speed_rpm)*10)`，上限 0xFFFF。
  - `emm_v5`：`addr + F6 + 方向 + 速度u16(RPM) + 加速度u8 + 同步 + 6B`。`speed_u16=round(abs(speed_rpm))`，accel 0..0xFF。
- **位置模式** `cmd_pos_mode`：`addr + FD + 方向 + 速度u16 + 加速度u16 + 脉冲u32 + 相对/绝对(0x00/0x01) + 同步 + 6B`。

#### 回包解析

- `verify_tail`：长度≥4 且末字节==0x6B。
- `parse_status_code`：兼容两种 ACK 格式（`addr cmd 状态 6B` 或 `addr cmd 功能字节 状态 6B`）；`frame[1]==0x00 且 frame[2]==0xEE` 抛“命令错误(0xEE)”；返回 `frame[-2]`（状态字节）。
- `parse_read_speed`：`addr 35 符号 速度u16 6B`。`parse_read_position_x10_deg`：`addr 36 符号 位置u32 6B`，角度×10。`parse_read_version`：`addr 1F 固件u16 硬件u16 6B`。

### 文件：`motor/controller.py`（高层控制器）

`ZDTMotorController` 组合 `ZDTV2Protocol` + `UARTBus`。

- **`MotorLimits`**：`max_speed_rpm=3000.0`, `max_accel=0xFFFF`, `pulses_per_rev=3200`（16 细分×200 步）。
- **构造**：`address=0x01`, `serial_config`, `limits`, `protocol_variant="zdt_v2"`。

| 方法 | 行为 | ACK 读取 |
|------|------|----------|
| `read_version()` | 0x1F | `exchange_min(min_size=7, timeout=0.8)` |
| `enable(enabled, sync, wait_ack)` | 0xF3 | `min_size=4` |
| `stop(wait_ack)` | 0xFE | `min_size=4` |
| `run_speed(direction, speed_rpm, accel=10, ...)` | 0xF6（校验 speed/accel 范围） | `min_size=4` |
| `move_relative(direction, speed_rpm, accel, pulses, ...)` | 0xFD（相对） | `min_size=4` |
| `read_speed_rpm()` | 0x35 | `min_size=6` |
| `read_position_x10_deg()` | 0x36 | `min_size=8` |
| `read_status_raw()` | 0x3A | `min_size=4` |

**`_check_ack`**：状态 `0x02`=成功；`0xE2`=条件不满足、`0xEE`=命令错误。使能命令若解析失败会尝试用命令字 `0x48` 再解析一次（某些固件使能回包命令字固定为 0x48）。读超时统一 0.8 s。

### 文件：`motor/kctrl.py`（键盘实时控制 CLI）

终端 raw 模式键盘控制（`termios`+`tty.setraw`，逐字符 `_getch`）。按键：`q`=正转、`e`=反转、`1`=减速、`2`=加速、`空格`=急停、`x`/`Ctrl+C`=退出。`_resolve_config_path()` 向上遍历找 `config/motor.yaml`。退出时 `finally` 中 `stop` + `enable(False)` + `close()`。

### 文件：`motor/conveyor_cli.py`（单发传送带命令，供 Qt/云调用）

一次性执行 `forward`/`reverse`/`stop`，参数 `--speed-ms`（m/s）。

#### m/s → RPM 换算算法（`_speed_ms_to_rpm`，优先级）

1. 配置 `rpm_per_ms`：`rpm = speed_ms * rpm_per_ms`。
2. 否则 `roller_diameter_m`：`rpm = speed_ms * 60 / (π * diameter)`（线速度→辊轴转速）。
3. 否则线性映射：`ratio=(speed_ms-min_ms)/(max_ms-min_ms)`, `rpm=min_rpm+ratio*(max_rpm-min_rpm)`。

流程：`_ensure_serial_port`（校验 `/dev/ttyS9`）→ clamp → `enable(True)` → `sleep(enable_delay_s)` → `run_speed(dir)`。stop 分支额外 `enable(False)`。

### 文件：`motor/diagnose_serial.py`（串口探测/诊断，不转电机）

- 默认波特率集 `(115200,57600,38400,25000,19200,9600)`；读命令 `{version:0x1F, speed:0x35, position:0x36, status:0x3A}`；ZDT 固定读帧 `01 43 7A 6B`。
- **三种校验**：`fixed`→恒 0x6B；`xor`→逐字节异或；`crc8`→**CRC-8/MAXIM**（初值=首字节，多项式 `0x8C` 反射右移）。
- `_classify` 返回 `no_rx/echo_only/echo_plus_reply/echo_plus_unknown/reply/driver_format_error/unknown_rx`。
- `--scan-ports` 扫 `/dev/ttyS* ttyUSB* ttyACM* ttyAMA*`；`--show-motion-frame` 打印使能/速度/停止帧；`--raw-zdt-status` 发一次状态帧。串口 `timeout=0.02, write_timeout=0.2`。

### 配置项 `config/motor.yaml`

- `serial`: `port:/dev/ttyS9`, `baudrate:115200`, `timeout_s:0.8`, `write_timeout_s:0.8`。
- `driver`: `address:"0x01"`, `protocol:emm_v5`（当前启用 Emm_V5）, `sync:0`。
- `kctrl`: `default_speed_rpm:1000.0`, `speed_step_rpm:10.0`, `min/max_speed_rpm:10.0/3000.0`, `accel:10`, `enable_delay_s:0.08`, `forward/reverse_direction:0/1`, `wait_ack:true`。
- `conveyor`: `min/max/default_speed_ms:0.1/0.18/0.13`, `slow/medium/fast:0.10/0.13/0.16`, `min/max_speed_rpm:10.0/3000.0`；可选 `rpm_per_ms:4000.0` / `roller_diameter_m:0.06`。

## sensors 模块（环境传感器）

包内含 4 个独立传感器驱动 + 统一监控器 `EM.py` + CSV 记录器 `csv_logger.py` + 权限脚本。所有 I2C 驱动共用常量 `I2C_SLAVE = 0x0703`，配置路径 `config/sensors.yaml`。**依赖库**：`fcntl`、`os`、`pyyaml`（无 smbus）。

### 文件：`sensors/sht30.py`（温湿度）

- **接口**：I2C bus 4（`/dev/i2c-4`），地址 `0x44`。
- **命令**：`SHT30_MEASURE_HIGHREP = bytes((0x24,0x00))`（高重复性、无时钟拉伸单次测量）。
- **流程** `_read_frame`：写 `0x2400` → `sleep(measure_delay_s=0.02)` → 读 6 字节（温MSB/LSB/CRC + 湿MSB/LSB/CRC）。
- **CRC** `_crc8`：初值 `0xFF`，多项式 `0x31`，左移（CRC-8/SENSIRION）。温、湿分别校验。
- **换算**：`temperature_c = -45.0 + 175.0*raw_t/65535.0`；`humidity_rh = clamp(100.0*raw_h/65535.0, 0..100)`。
- `read_with_retry(max_retries, retry_interval_s)`。**异常** `SHT30Error`。

### 文件：`sensors/scd40.py`（CO₂/温/湿）

- **接口**：I2C bus 4，地址 `0x62`。
- **命令码（u16，高字节先发）**：`START_PERIODIC=0x21B1`、`STOP_PERIODIC=0x3F86`、`GET_DATA_READY=0xE4B8`、`READ_MEASUREMENT=0xEC05`。`DEFAULT_MEASUREMENT_PERIOD_S=5.0`。
- **CRC** 同 SHT30。`data_ready()`：读 3 字节→CRC→`status=(b0<<8)|b1`，`(status & 0x07FF)!=0` 表示有新数据。
- `read_once()`：读 9 字节（3 组 `2B数据+1B CRC`，偏移 0/3/6）。`co2_ppm=words[0]`；`temperature_c=-45+175*words[1]/65535`；`humidity_rh=clamp(100*words[2]/65535,0..100)`。
- **预热**：`warmup_s=5.0`（配置 6.0 s ready_timeout）；stop 后 `sleep(0.5)`。**异常** `SCD40Error`。

### 文件：`sensors/gy302.py`（GY-302 / BH1750 光照）

- **接口**：I2C bus 4，地址 `0x23`。
- **模式表** `GY302_MODES`：`continuous_high_res_1=0x10`、`_2=0x11`、`continuous_low_res=0x13`、`one_time_high_res_1=0x20`、`_2=0x21`、`one_time_low_res=0x23`。默认 `one_time_high_res_1`。
- **流程** `_read_raw`：写单字节模式码 → `sleep(measure_delay_s=0.18)` → 读 2 字节 → `raw=(b0<<8)|b1`。
- **换算**：`lux = raw / 1.2`（BH1750 标准系数）。**异常** `GY302Error`。

### 文件：`sensors/mq135.py`（MQ-135 空气质量，经 SARADC/IIO）

- **接口**：sysfs IIO，`iio_device="iio:device0"`，通道 `channel=4`（VIN4）。路径：`/sys/bus/iio/devices/iio:device0/in_voltage4_raw`、`.../in_voltage_scale`、`.../name`。
- **算法** `read_once`：
  - `voltage_mv = raw * scale_mv`
  - `level_pct = clamp(raw/adc_raw_max*100, 0..100)`（`adc_raw_max=4095`）
  - `Rs = RL*(Vref - V)/V`（`RL=10kΩ`, `Vref=1800mV`；V≤0 时 Rs=∞）
  - `ratio = Rs / R0`（`R0=76.63kΩ`）
  - **`ppm_est = gas_curve_a * ratio^gas_curve_b * ppm_scale`** = `116.6020682 * ratio^(-2.769034857) * 0.01`（幂律经验曲线）。
- 返回 `raw, scale_mv, voltage_mv, level_pct, rs_kohm, ratio, ppm_est`。**异常** `MQ135Error`。

### 文件：`sensors/EM.py`（统一环境监控器 EnvironmentMonitor）

聚合 SHT30/SCD40/GY302/MQ135，统一初始化/读取/渲染（用 Tab 缩进）。各传感器 `enabled` 可控。初始化独立 try/except（失败记入 `_init_errors` 不整体崩溃），SCD40 初始化后立即 `start_periodic_measurement()`。`read_all()` 返回每传感器 `{"ok":True,...}`/`{"ok":False,"error":..}`/`{"ok":None,"disabled":True}`。CLI：`--count`、`--interval`、`--fail-fast`。

### 文件：`sensors/csv_logger.py`（实时 CSV 记录器）

- **输出**：`datas/csv/sensor_realtime.csv`。
- **CSV 列** `CSV_FIELDS`：`temperature_c, humidity_rh, co2_ppm, light_lux, air_quality_ppm, env_status, timestamp, sensor_errors`。
- **映射** `_build_row`：温湿度优先 SHT30 回退 SCD40；`co2_ppm` 取 SCD40；`light_lux` 取 gy302.lux；**`air_quality_ppm` 实际写 MQ135 的 `level_pct`（相对水平%）**。
- **CO₂ 缓存**：`co2_cache_max_age_s`（默认 20 s）内 SCD40 无新数据复用缓存且不计 error。
- `env_status`："正常"/"部分异常"；`sensor_errors`：`"name:error; ..."`。
- **写盘**：表头 `_ensure_header`（`fsync`）；每行 `flush + os.fsync`（保证 Qt 立即可读）。
- **生命周期**：`SIGTERM/SIGINT` 停止；`--parent-pid` 监测父进程存活，消失即退出（0.2 s 粒度轮询）。

### 文件：`sensors/setup_sensor_permissions.sh`（硬件权限总配置，需 sudo）

全硬件层 udev 权限与用户组配置（LED 脚本也转发到此）：
- 创建组 `i2c spi gpio pwm`，用户加入 `i2c,spi,gpio,pwm,dialout,video,input`。

- `udevadm control --reload-rules` + trigger，并对已存在节点立即 chgrp/chmod 0660。

## servo 模块（舵机分拣拨杆）

包 `__init__.py` 导出 `PWMConfig, ServoCalibration, ServoDriver, ServoError, SorterServo, SorterServoError, load_sorter_servo`。

### 文件：`servo/driver.py`（PWM sysfs 舵机底层驱动）

**依赖库**：仅标准库（`errno`, `pathlib`, `time`）。**异常** `ServoError`。

#### 硬件接口（PWM sysfs）

`PWMConfig`：`chip=2`（代码默认，YAML 覆盖为 0）、`channel=0`、`base_path=/sys/class/pwm`、`polarity="normal"`。派生路径：`chip_path=/sys/class/pwm/pwmchip{chip}`、`pwm_path=chip_path/pwm{channel}`；操作文件 `export`、`period`、`duty_cycle`、`enable`、`polarity`。

#### 舵机标定 `ServoCalibration`

`period_ns=20_000_000`（20 ms=50 Hz）、`min_pulse_us=500`、`neutral_pulse_us=1500`、`max_pulse_us=2500`、`min_deg=-90`、`neutral_deg=0`、`max_deg=90`。`validate()`：period>0；脉宽 `0<min<neutral<max`；角度 `min<neutral<max`；`max_pulse_us*1000 < period_ns`。

#### 角度↔脉宽双线性映射算法

```
angle ≥ neutral:  pulse = neutral_pulse + (angle-neutral)/(max_deg-neutral) * (max_pulse-neutral_pulse)
angle < neutral:  pulse = neutral_pulse - (neutral-angle)/(neutral-min_deg) * (neutral_pulse-min_pulse)
```
`pulse_us_to_angle` 为逆运算；`clamp_angle` 限幅到 `[min_deg,max_deg]`。

#### `ServoDriver` 方法与时序

| 方法 | 行为 |
|------|------|
| `open()` | validate → 若 `pwm_path` 不存在则写 `export` 并 `_wait_for_pwm_path`(1s) → 设 polarity → `_write_period_safely` |
| `enable()` | open → 写 `enable=1` |
| `set_angle(angle)` → `set_pulse_us` | 限幅 → `duty_ns` → 写 `duty_cycle` → 未使能则 enable |
| `close(disable=True)` | 可选 disable |

- `_pulse_us_to_duty_ns`：`high_ns=pulse_us*1000`；极性 `inversed` 时 `duty=period_ns-high_ns`。
- `_write_period_safely`：period 已相符则跳过；否则先关 enable 再写 period 再恢复。
- `_write_file`：`export` 遇 `EBUSY` 且通道已存在则静默返回；极性写失败仅告警。

### 文件：`servo/sorter.py`（分拣舵机高层 + CLI）

- `SorterPositions`：`center_deg=0`, `left_deg=-45`, `right_deg=45`。
- `SorterServo(driver, positions, settle_s=0.35)`：`move_to` → `set_angle` + 可选 `sleep(settle_s)`；`sort_to(position)` 归一化别名：
  - left: `{"left","-45","1","pos1","position1"}`
  - center: `{"center","middle","0","2","pos2","position2"}`
  - right: `{"right","45","3","pos3","position3"}`
- `load_sorter_servo(config_path)`：读 `servo:` 的 `pwm/timing/angle/sorter` 子块。
- **CLI**：位置参数 `1/2/3/left/center/right/-45/0/45`；`--config`、`--no-wait`、`--disable-after-move`（默认移动后保持 PWM 使能以锁位）。

### 文件：`servo/test_servo.py`（键盘测试 + PWM 诊断）

- `--diagnose`：遍历 `/sys/class/pwm/pwmchip*`，打印 `npwm`、`DRIVER=`/`OF_FULLNAME=`、各 `pwmN` 的 `period/duty_cycle/enable/polarity`。
- 交互（`termios` raw）：`Q/q=-step(30°)`、`E/e=+step`、`0=center`、`A=left(-45°)`、`D=right(45°)`、`X/x=exit`。

## spectrum 模块（AS7341 光谱颜色/成熟度分析）

三文件：`AS7341.py`（驱动）、`color.py`（颜色分析）、`collect.py`（数据集采集）。配置读取 `sensors.yaml` 的 `sensors.as7341` 段。

### 文件：`spectrum/AS7341.py`

**作用**：驱动 AMS AS7341 8 波段可见光 + Clear + NIR 光谱传感器。**依赖库**：`fcntl`, `os`, `pyyaml`。**异常** `AS7341Error`。

#### 硬件接口

- I2C bus 7（`/dev/i2c-7`），地址 `0x39`，`I2C_SLAVE=0x0703`。
- **寄存器**：`WHOAMI=0x92`、`ENABLE=0x80`、`ATIME=0x81`、`CFG0=0xA9`、`CFG1=0xAA`、`CFG6=0xAF`、`STATUS2=0xA3`、`ASTATUS=0x94`、`ASTEP_L=0xCA`（+1=ASTEP_H）。
- **ENABLE 位**：`PON=0x01`、`SP_EN=0x02`、`SMUXEN=0x10`。`STATUS2 AVALID=0x40`。
- **芯片 ID**：`EXPECTED_DEVICE_ID=0x09`，`chip_id=(WHOAMI>>2)&0x3F`。

#### 增益/积分参数

- `GAIN_TO_CODE`：0.5→0, 1→1, 2→2, 4→3, 8→4, 16→5, 32→6, 64→7, 128→8, 256→9, 512→10（写 CFG1 低 5 位）。
- **积分时间**：`integration_time_ms = (atime+1)*(astep+1)*2.78e-3`（atime=100,astep=999 → ≈280.5 ms）。

#### SMUX 多路复用（8 通道映射到 6 ADC，分两次读）

AS7341 只有 6 个 ADC，需 SMUX 两轮（F1–F4 + F5–F8，各带 Clear/NIR）：`SMUX_F1F4_CLEAR_NIR` 与 `SMUX_F5F8_CLEAR_NIR` 各 20 条 `(reg,out1,out2)`，`packed=(out1&0x0F)|((out2&0x0F)<<4)` 写 RAM bank0。`_trigger_smux_command`：`CFG6[4:3]=0b10` → 置 SMUXEN → 轮询自清。`_wait_data_ready`：轮询 `STATUS2 & AVALID`。

#### 读取流程

- `initialize()`：校验 WHOAMI → 置 PON → sleep 10 ms → 写 ATIME、ASTEP（0xCA/0xCB）、CFG1(gain)。
- `_read_adc6()`：读 `ASTATUS` 起 13 字节，每通道小端 u16。
- `read_once()`：f1f4 读 low[6]，f5f8 读 high[6]：`f1_415=low[0]…f4_515=low[3]`、`f5_555=high[0]…f8_680=high[3]`、`clear=round((low[4]+high[4])/2)`、`nir=round((low[5]+high[5])/2)`；另含 low/high 分量、`integration_time_ms`、`gain`。
- CLI 打印 `F1..F8 CLEAR NIR`。默认 `bus=7, addr=0x39, atime=100, astep=999, gain=128`。

### 文件：`spectrum/color.py`（光谱→颜色/CIE/HSV）

**依赖库**：`colorsys`, `json`。

- `WAVELENGTHS_NM`：f1_415→415 … f8_680→680。
- `CIE1931_2DEG_XYZ`：8 波长的 CIE 1931 2° 配色函数（如 555nm=(0.5121,1.0000,0.0057)）。

**算法**：
1. `_calibrate_bands`：`value=max(0,raw-dark)`；有白参考 `value/=max(1e-6,white-dark)`。
2. `_compute_xyz`：`X=Σ v·xb`（Y、Z 同），归一化 `x=X/(X+Y+Z)`。
3. `_xyz_chromaticity_to_srgb`：D65 XYZ→线性 sRGB 矩阵：
   ```
   r=3.2406X-1.5372Y-0.4986Z; g=-0.9689X+1.8758Y+0.0415Z; b=0.0557X-0.2040Y+1.0570Z
   ```
   出色域加白去饱和；归一化到 max；sRGB gamma（≤0.0031308 用 12.92c，否则 `1.055c^(1/2.4)-0.055`）；×255 限幅。
4. `_compute_energy`：`blue=f1+f2+f3`, `green=f4+f5`, `red=f6+f7+f8`。
5. `analyze_spectrum`：主导波段、`spectral_centroid_nm=Σ(nm·v)/Σv`、`peak_ratio=dominant/total`、RGB→HSV、色名、`confidence=clip(0.55·s + 0.45·min(1,peak_ratio·3),0..1)`、`nir_ratio=nir/max(clear,1)`。

**`_name_from_hsv` 颜色判定**：v<0.08→black；s<0.12→white/gray/dark_gray；15≤h<45 且 v<0.65→brown；否则 hue 分段 red/orange/yellow/green/cyan/blue/purple/magenta；兜底 centroid≥600→warm。

**输出** dict：`dominant_band, dominant_wavelength_nm, dominant_value, spectral_centroid_nm, cie_xy, srgb, hsv, color_name, confidence, energy, calibrated, nir_ratio`。

### 文件：`spectrum/collect.py`（数据集采集）

**输出**：`datas/spectrum_quality_samples.csv`（`--output` 可改；`--append` 追加）。

**CSV 列（60 列）**：`timestamp_iso, timestamp_ms, sample_index, sample_id, fruit_type_manual, fruit_variety_manual, yolo_fruit_type, yolo_fruit_type_conf, yolo_maturity, yolo_maturity_conf, yolo_defect, yolo_defect_conf, yolo_bbox_xyxy, fruit_type_final, maturity_final, defect_final, quality_grade, brix_label, quality_label, note, sensor_bus, sensor_address, atime, astep, gain, integration_time_ms, f1_415, f2_445, f3_480, f4_515, f5_555, f6_590, f7_630, f8_680, clear, nir, nir_ratio, dominant_band, dominant_wavelength_nm, dominant_value, spectral_centroid_nm, cie_x, cie_y, srgb_r, srgb_g, srgb_b, hsv_h_deg, hsv_s, hsv_v, color_name, confidence, red_sum, green_sum, blue_sum, calib_white_ref, calib_dark_ref, raw_json, analysis_json`。

**标签融合**：
- `_pick_final(yolo, manual, unknown)`：优先 YOLO，其次人工，都空取 unknown（defect 兜底 `"none"`）。
- `_infer_quality_grade`：有 `quality_label` 直接用；defect ∈ 缺陷集 → `reject`；maturity ∈ 未熟集 → `not_ready`；成熟且 brix≥14→`premium`,≥12→`good`,否则 `normal`；ripe 无 brix→`normal`；其余 `unknown`。
