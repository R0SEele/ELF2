# 03 · 芒果质检核心算法技术文档

## 概述

核心品质检测算法采用**基于规则的专家系统（rule-based expert baseline）**，无需训练数据集，通过融合三路信号得出品质结论：

1. **视觉信号（vision）**：YOLO11（RKNN 量化模型 `mango_yolo11_rk3588_i8.rknn`）对芒果的检测标签与置信度，以及从检测 ROI 提取的 RGB/HSV 颜色特征。
2. **光谱信号（spectral）**：AS7341 8 通道可见光光谱传感器（F1~F8）加 CLEAR/NIR 通道，经色度学处理得到红/绿/黄光谱比、光谱质心、NIR 比等衍生特征。
3. **传感器融合（fusion）**：将上述信号加权组合，输出成熟度、糖度参考、腐烂风险、品质等级与建议分拣通道。

整体数据流：

```
摄像头 --> YOLO11(RKNN) --> vision_color_realtime.csv / mango_object_realtime.csv
                                              |
AS7341 传感器 --> collect.py/color.py --> spectrum_quality_samples.csv
                                              |
                                              v
                        fusion.assess_mango_quality()  <-- 核心融合算法
                                              |
              +-------------------------------+-------------------------------+
              v                               v                               v
   mango_quality_realtime.csv/json   mango_batch_summary.*        AutoSorter.sort_once()
   mango_quality_history.csv                                              |
                                                              sorter.py <position> --> 舵机 PWM 分拣
                                                              mango_sorter_events.csv
```

> README 明确说明：**无实测 Brix 数据集，糖度输出为参考区间而非标定测量值**；若视觉 CSV 缺少斑点/褐变面积字段，腐烂判断会退化为基于 ROI 均值 HSV/RGB 的弱判断。
> `config/yolo11.yaml` 仅描述 YOLO11 检测运行参数，不直接引用品质逻辑；其类别标签 `mango_unripe/mango_ripe/mango_overripe` 被融合算法消费。

## 一、视觉颜色特征提取 — `color_features.py`

### `extract_mango_color_features_from_bgr_roi(roi_bgr)`

**输入**：BGR ROI ndarray。空图返回 `{}`。

**色彩空间转换**：
- `cv2.cvtColor(..., COLOR_BGR2HSV)`。OpenCV H 范围 0~179，故 **`hue_deg = H*2.0`**；`sat_pct = S*100/255`；`val_pct = V*100/255`。
- R/G/B 取 `roi_bgr[:,:,2]/[:,:,1]/[:,:,0]`。

**有效像素掩码**（排除背景/阴影）：`valid = (sat_pct>12.0) & (val_pct>10.0)`；`total=max(1, valid.sum())`。

**四类颜色掩码与阈值（load-bearing 常量）**：
- 绿色：`valid & (35.0≤hue≤95.0) & (sat≥20.0)`
- 黄橙：`valid & (8.0≤hue≤38.0) & (sat≥25.0)`
- 暗斑：`(val<28.0) | ((r<65)&(g<65)&(b<65))`
- 褐变：`(8.0≤hue≤42.0) & (sat≥18.0) & (val≤55.0) & (r≥g*0.85) & (g≥b*0.85)`

**输出字段**：`rgb_r/g/b_mean`（ROI 均值）、`hsv_h_mean_deg`（H均值×2）、`hsv_s_mean_pct`、`hsv_v_mean_pct`、`green_ratio=green.sum()/total`、`yellow_orange_ratio=yellow.sum()/total`、`dark_spot_ratio=dark.sum()/(H*W)`（**全部像素**为分母）、`brown_area_ratio=brown.sum()/total`。

> 注意 `dark_spot_ratio` 用图像总面积做分母，其余比值用有效像素数做分母。

## 二、光谱采集与分析（融合输入）

### AS7341 关键接口

- 默认 `bus=7, addr=0x39, atime=100, astep=999, gain=128.0`；积分时间 `(atime+1)*(astep+1)*2.78e-3`。
- 通道波长：F1=415, F2=445, F3=480, F4=515, F5=555, F6=590, F7=630, F8=680 nm。

### color.py 派生特征（融合关注）

- 能量分区：`blue=f1+f2+f3`, `green=f4_515+f5_555`, `red=f6_590+f7_630+f8_680`。
- `spectral_centroid_nm = Σ(波长·校正值)/Σ校正值`。
- `nir_ratio = nir/max(clear,1.0)`。
- `confidence = clip(0.55·s + 0.45·min(1,peak_ratio·3),0,1)`。

### collect.py `_infer_quality_grade`（采集端英文等级，供数据集标注，与 fusion 中文等级不同）
- 缺陷集 → `reject`；未熟集 → `not_ready`；成熟且 brix≥14→`premium`,≥12→`good`,否则 `normal`；成熟无 brix→`normal`；其余 `unknown`。

## 三、核心融合算法 — `fusion.py`

### 3.1 路径常量与陈旧阈值

```
DEFAULT_VISION_CSV   = datas/csv/vision_color_realtime.csv
DEFAULT_OBJECT_CSV   = datas/csv/mango_object_realtime.csv
DEFAULT_SPECTRUM_CSV = datas/spectrum_quality_samples.csv
ALT_SPECTRUM_CSV     = datas/csv/spectrum_realtime.csv
DEFAULT_OUTPUT_CSV   = datas/csv/mango_quality_realtime.csv
DEFAULT_OUTPUT_JSON  = datas/csv/mango_quality_realtime.json
DEFAULT_BATCH_CSV    = datas/csv/mango_batch_summary.csv
DEFAULT_BATCH_JSON   = datas/csv/mango_batch_summary.json
DEFAULT_HISTORY_CSV  = datas/csv/mango_quality_history.csv
OBJECT_STALE_SECONDS = 20.0
```

### 3.2 标签映射 `MANGO_LABELS`（YOLO 标签 → 内部键, 成熟度基准分, 中文名）

- `mango_unripe / unripe / 芒果-未熟 / 未熟` → `("unripe", 20.0, "未熟")`
- `mango_ripe / ripe / 芒果-成熟 / 成熟` → `("ripe", 65.0, "成熟")`
- `mango_overripe / overripe / 芒果-过熟 / 过熟` → `("overripe", 88.0, "过熟")`

即 **YOLO 标签直接给出成熟度基准分：未熟=20、成熟=65、过熟=88**。

### 3.3 通用数学工具

- `clamp(v, low=0, high=100)`：截断，NaN 返回 low。
- `ratio_score(v, low, high) = clamp((v-low)*100/(high-low))`。
- `inverse_ratio_score(v, low, high) = 100 - ratio_score(...)`。
- `weighted_average(items)`：`(值,权重)` 列表，跳过 NaN 或权重≤0，返回 `Σ(值·权)/Σ权`。

### 3.4 数据读取

- **`read_latest_vision(path)`**：仅保留 `MANGO_LABELS` 可识别的行，按 `rank = confidence * max(1.0, log10(max(area_px,1.0)))` 选最高分行。
- **`read_latest_object(path)`**：要求时间戳在 `OBJECT_STALE_SECONDS=20s` 内，否则 None。CLI 优先用 object，回退 vision。
- **`read_latest_spectrum(path)`**：取 CSV 最后一行。派生：`green_sum` 缺失=`f4+f5`；`red_sum` 缺失=`f6+f7+f8`；`blue_sum` 缺失=`f1+f2+f3`；`denominator=max(green_sum,1.0)`；**`red_green_ratio=red_sum/denominator`**；**`yellow_green_ratio=f6_590/denominator`**；`nir_ratio` 缺失=`nir/max(clear,1.0)`。

### 3.5 成熟度评分（三个独立评分器）

**(a) YOLO 成熟度分**：直接返回标签映射分（20/65/88）。

**(b) 颜色成熟度分 `color_maturity_score`**：优先用面积比加权（锚点分×面积权）：
```
weighted_average([(20.0,green_ratio),(70.0,yellow_orange_ratio),(88.0,brown_area_ratio),(92.0,dark_spot_ratio)])
```
面积比全缺失时退化为 HSV 色相分段：
- `35≤hue≤95`：`22.0 + inverse_ratio_score(hue,35,95)*0.12`
- `10≤hue<35`：`63.0 + inverse_ratio_score(hue,10,35)*0.12`
- `hue<10 或 hue≥330`：`78.0`；其余：`55.0`
再叠加：`sat` 有效 `+= ratio_score(sat,20,75)*0.08`；`val<35 → +14.0`，`val>70 → -4.0`。

**(c) 光谱成熟度分 `spectrum_maturity_score`**：
```
rg_score       = ratio_score(red_green_ratio, 0.85, 2.35)
yg_score       = ratio_score(yellow_green_ratio, 0.15, 1.10)
centroid_score = ratio_score(spectral_centroid_nm, 510, 640)
nir_score      = ratio_score(nir_ratio, 0.02, 0.45)
分值 = weighted_average([(rg,0.45),(yg,0.20),(centroid,0.25),(nir,0.10)])
```

### 3.6 腐烂/褐变/暗斑评分

- **`estimate_dark_score`**：有 `dark_spot_ratio` 时 `clamp(dark_spot_ratio*280.0)`；否则 `inverse_ratio_score(val,25,70)`（权 0.65）与 `inverse_ratio_score(rgb均值,45,155)`（权 0.35）加权。
- **`estimate_brown_score`**：有 `brown_area_ratio` 时 `clamp(brown_area_ratio*240.0)`；否则色相 `8≤hue≤38→100`、`38<hue≤55→inverse_ratio_score(hue,38,55)`，与 `ratio_score(sat,15,65)`（权 0.20）、`inverse_ratio_score(val,35,75)`（权 0.35）加权，色相信号权 0.45。
- **`spectrum_abnormal_score`**：
  ```
  high_red=ratio_score(red_green_ratio,1.9,3.2); high_centroid=ratio_score(centroid,610,680); low_nir=inverse_ratio_score(nir_ratio,0.03,0.35)
  = weighted_average([(high_red,0.45),(high_centroid,0.35),(low_nir,0.20)])
  ```
- **`rot_score`**（腐烂综合）：`overripe_signal=confidence*100`（label==overripe 时）。动态权重：`dark 0.45(有)/0.15`、`brown 0.25(有)/0.35`、`overripe 0.25(>0)/0`、`spectrum 0.15(有)/0`。

### 3.7 糖度评分 `sugar_score`

```
sat_score      = ratio_score(hsv_s_mean_pct, 20, 75)                          (vision 有效)
spectrum_sugar = weighted_average([(ratio_score(red_green_ratio,0.85,2.20),0.65),(ratio_score(nir_ratio,0.02,0.45),0.35)])
score = weighted_average([(maturity_score,0.50),(spectrum_sugar,0.30),(sat_score,0.20)])
若 rot_value ≥ 65.0: score = min(score, 82.0)   # 腐烂封顶
return clamp(score)
```

### 3.8 标签化（阈值→中文标签）

- **成熟度** `maturity_label`：`<45→未熟`，`<78→成熟`，`≥78→过熟`，NaN→无法评估。
- **糖度** `sugar_label_and_brix`：`rot≥65 → ("过熟异常，糖度参考不可靠","不建议参考")`；`<40 → ("偏低","8-12 Brix")`；`<70 → ("中等","12-16 Brix")`；`<85 → ("较高","16-20 Brix")`；否则 `("过熟偏高，仅供参考","18-22 Brix")`。
- **腐烂** `rot_status`：`<35→正常`，`<65→疑似腐烂`，`≥65→腐烂/建议剔除`。
- **最终状态** `final_status`：腐烂→建议剔除；疑似→需要复检；成熟度无效→无有效检测；否则→可接受。

### 3.9 品质等级 `quality_grade`（按优先级）

1. 腐烂/建议剔除 → **剔除**
2. 疑似腐烂 → **C级**
3. 成熟 且 sugar∈{较高,中等} → **A级**
4. 成熟（其余糖度） → **B级**
5. 未熟/过熟 → **B级**
6. 否则 → **--**

### 3.10 建议分拣通道 `suggested_channel`

1. 腐烂/建议剔除 → **剔除通道**
2. 疑似腐烂 → **复检通道**
3. 未熟 → **催熟通道**
4. 成熟 → **销售通道**
5. 过熟 → **加工通道**
6. sugar 无法评估 → **等待数据**
7. 否则 → **销售通道**

### 3.11 顶层融合 `assess_mango_quality(vision, spectrum)`

1. 无 vision → 返回“未检测到芒果”空评估。
2. 计算 `yolo_score / color_score / spectral_score`。
3. **成熟度加权融合（核心权重）**：
   ```
   yolo_weight = 0.60 * max(0.25, vision.confidence)     # 下限 0.25×0.60=0.15
   maturity_value = weighted_average([(yolo_score, yolo_weight), (color_score, 0.25), (spectral_score, 0.15)])
   ```
   即 YOLO 满置信度时权重 0.60，颜色固定 0.25，光谱固定 0.15。
4. `rot_value` → `sugar_value`。
5. 标签化 → `grade` → `channel`。
6. **成熟度置信度**：`source_count=(vision存在)+(spectrum存在)`；`maturity_confidence=clamp(45.0 + confidence*35.0 + source_count*10.0)`；无光谱时 `min(该值,78.0)`。
7. 组装 `QualityAssessment`（成熟度/腐烂/糖度 2 位小数，置信度/一致性 4 位），携带原始特征与 `reason`（`build_reason` 人读说明串）。

## 四、自动分拣 — `auto_sorter.py` + `servo.yaml` + `sorter.py`

### 4.1 `AutoSorter`

配置来源 `config/servo.yaml > servo.auto_sort`：`enabled`、`delay_s=0.8`（决策到执行延迟）、`reset_to_center=true`、`reset_delay_s=0.45`、`default_position="2"`、`log_csv`、`channel_positions`、`grade_positions`。启用时启动守护线程 `mango-auto-sorter` 消费队列。

**`position_for(assessment)` 优先级**：① `suggested_channel` 命中 `channel_positions` → 用之；② 否则 `quality_grade` 命中 `grade_positions`；③ 否则 `default_position`（"2"）。

**`sort_once`**：未启用返回 False；`mango_id` 空/`--`/已分拣（`sorted_ids` 去重）则跳过；否则计算位置、入队。

**`_execute_sort`**（工作线程）：`sleep(delay_s)` → `python3 sorter.py <position> --config servo.yaml`（cwd=PROJECT_ROOT，timeout=5s）→ 记录 `result` + 截断 180 字符 message 追加写事件 CSV → 若成功且 `reset_to_center` 且 `position!="2"`：`sleep(reset_delay_s)` 后 `sorter.py 2` 复位。

### 4.2 品质等级 → 物理料箱映射链

```
成熟 --> 销售通道 --> "2" --> 旋转360°
未熟 --> 催熟通道 --> "1" --> 旋转360°
过熟 --> 加工通道 --> "3" --> 旋转360°
疑似腐烂 --> 复检通道 --> "3" --> 旋转360°
腐烂 --> 剔除通道 --> "3" --> 旋转360°
```
三个档位值仅保留上层通道映射兼容性，底层不再定位。每条命令都以 `2500 us` 旋转 `revolution_duration_s`，随后用 `1500 us` 停止；不记录当前位置。

## 九、设计要点与局限

1. **多源鲁棒性**：任一信号缺失仍可运行——YOLO 权重随置信度收缩、颜色/光谱评分 NaN 时被 `weighted_average` 跳过；缺光谱时成熟度置信度封顶 78。
2. **腐烂优先**：分级与通道逻辑腐烂优先级最高，可直接触发剔除/复检。
3. **糖度为参考**：无 Brix 标定集，`reference_brix_range` 仅供显示；腐烂时标注“不可靠/仅供参考”。
4. **原子写 + 去重**：输出用 `.tmp`+`os.replace`；历史/批次按 `mango_id` 去重，分拣按 `mango_id` 去重防重复动作。
5. **弱化退路**：视觉 CSV 缺 `dark_spot_ratio/brown_area_ratio` 时退化到 ROI 均值 HSV/RGB 判断；`extract_mango_color_features_from_bgr_roi()` 用于写视觉 CSV 前补齐这些字段。
